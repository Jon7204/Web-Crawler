# Search Engine Tool

A command-line search engine built in Python that crawls [quotes.toscrape.com](https://quotes.toscrape.com), constructs an inverted index, and provides ranked search results with advanced query processing.

---

## Features

- **BFS Web Crawler** — Breadth-first crawl of all pages with a configurable politeness window
- **Inverted Index** — Stores word frequency and token positions per page
- **TF-IDF Ranking** — Results ranked by relevance, not just occurrence
- **Boolean Operators** — AND, OR, NOT query support
- **Stopword Filtering** — Common words filtered to reduce noise (configurable)
- **Query Suggestions** — Suggests similar words when a search term isn't found
- **Persistent Storage** — Index saved to and loaded from disk as JSON
- **Comprehensive CLI** — Built-in help system with per-command documentation

## Usage

Start the interactive shell from the project root:

```bash
python src/main.py
```

### Commands

| Command | Description |
|---|---|
| `build` | Crawl quotes.toscrape.com and build the inverted index |
| `load` | Load a previously built index from `data/index.json` |
| `print <word>` | Display the full index entry for a word |
| `find <query>` | Search for pages matching the query, ranked by relevance |
| `help` | Show all available commands and query syntax |
| `build --help` | Show help for the build command |
| `find --help` | Show help for the find command |
| `print --help` | Show help for the print command |
| `load --help` | Show help for the load command |
| `quit / exit / q` | Exit the tool |

### Examples


### Boolean Query Syntax

| Syntax | Behaviour |
|---|---|
| `find love` | Pages containing "love" |
| `find good friends` | Pages containing both "good" and "friends" |
| `find love AND friendship` | Intersection — pages containing both terms |
| `find love OR friendship` | Union — pages containing either term |
| `find love NOT death` | Pages with "love" but not "death" |

> **Note:** Boolean operators must be uppercase (AND, OR, NOT) to distinguish them from search terms. Chained boolean operators are not supported (e.g. `find love AND friendship OR hope`).

## Testing

Run the full test suite from the project root:

```bash
python -m pytest tests/ -v
```

Run with coverage report:

```bash
pip install pytest-cov
python -m pytest tests/ -v --cov=src --cov-report=term-missing
```

### Testing Strategy

All HTTP requests are mocked using `unittest.mock` — no network access is required to run the tests. The suite covers:

- **Crawler** — Link extraction, BFS traversal, deduplication, politeness window enforcement, graceful error handling (404, connection errors, timeouts)
- **Indexer** — Tokenisation, stopword filtering, frequency counting, position tracking, TF-IDF calculation, edge cases (empty pages, repeated words, case normalisation)
- **Search** — Single and multi-word queries, boolean operators (AND/OR/NOT), edge cases (empty queries, missing terms, malformed operators), query suggestions
- **CLI** — All command branches, load/save round-trip, error messages, help system

## Design Decisions

### Inverted Index Structure

```python
{
    "word": {
        "url": {
            "frequency": int,
            "positions": [int, ...]
        }
    }
}
```

A nested dictionary gives O(1) average-case lookup by word then by URL — efficient for both `print` and `find` operations. Positions are stored to support potential phrase-search extensions.

### Storage Format: JSON

The index is serialised as JSON for human readability and easy debugging. For very large corpora a binary format (e.g. SQLite) would be more efficient, but JSON is appropriate for this dataset and aids transparency.

### Crawling: BFS

Breadth-first search ensures nearby pages are visited before deep ones, which is more likely to surface important pages early and avoids getting stuck in deep link chains.

### Ranking: TF-IDF

Results are ranked by Term Frequency–Inverse Document Frequency:

- **TF** = raw frequency of the term in the document
- **IDF** = log(total documents / (1 + documents containing term))
- **Score** = TF × IDF

This weights terms that appear frequently in a document but rarely across the corpus, producing more relevant results than simple frequency counting.

### Stopword Filtering

Common words (e.g. "the", "a", "is") are filtered during tokenisation to reduce index noise. Filtering is controlled by the `STOPWORD_FILTERING` flag in `indexer.py` and can be disabled if needed.

### Boolean Operators

Operators must be uppercase to distinguish them from search terms — consistent with standard query languages (SQL, Elasticsearch). Compound queries (e.g. `love AND friendship OR hope`) are not supported; a single operator per query was chosen to prioritise correctness over incomplete complexity.

## Known Limitations

- Boolean operators do not support compound queries (e.g. `A AND B OR C`)
- TF-IDF scores can be negative when a word appears in a high proportion of pages — this is mathematically expected and indicates the word is not distinctive
- The trailing slash duplication (`quotes.toscrape.com` vs `quotes.toscrape.com/`) results in two index entries for the same page — a known limitation of URL normalisation
- Navigation and footer text is indexed alongside body content, which can inflate results for common words
